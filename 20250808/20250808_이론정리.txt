> Transaction
  - @Transactional
  - @EnableTransactionManagement
  1. **트랜잭션**: 여러 작업을 하나로 묶어 전부 성공 또는 전부 실패하도록 보장
  2. **ACID 특성**:
      - 원자성(전부/전무), 일관성(규칙 준수), 격리성(동시 실행 보호), 지속성(영구 저장)
  3. **커밋**: 모든 작업 성공 시 변경사항을 영구 저장
  4. **롤백**: 실패 시 모든 작업을 취소하고 원래 상태로 복구

  - 메서드 레벨 트랜젝션 : 특정 메서드에만 트랜잭션이 필요한 경우 사용
  - 클래스 레벨 트랜젝션 : 클래스의 모든 public 메서드에 트랜잭션 적용
  - 메서드 레벨 설정이 클래스 레벨보다 우선순위 높음
