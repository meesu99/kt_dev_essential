> 상속
  - 상속은 기존 클래스를 재사용하면서 새로운 기능을 추가할 수 있게 해주는 객체지향 프로그래밍의 핵심 기능
  - 자식 클래스가 부모 클래스의 속성(필드)과 메서드를 물려받아 사용
  * class 자식클래스 extends 부모클래스


> super 키워드
  - 부모 클래스의 생성자나 메서드를 참조할 때 사용


> 오버라이딩(Overriding)
  - 부모에게 상속받은 메서드를 자식 클래스에서 재정의
  * 메서드명, 매개변수, 반환타입이 부모와 동일해야 함
  * 접근제한자는 부모보다 좁아질 수 없음
  * @Override 애너테이션 사용 권장


> 오버로딩(Overloading)
  - 같은 이름의 메서드를 여러 개 정의
  * 메서드명은 동일해야 함
  * 매개변수의 개수 또는 타입이 달라야 함
  * 반환타입만 다르면 컴파일 오류
  * 접근제한자는 상관없음


> 인터페이스
  - 다중 구현이 필요할 때
  - 서로 다른 클래스들이 같은 기능 제공할 때
  - 느슨한 결합이 필요할 때
  - 인터페이스 내에서 키워드(default / static / private) 없이 사용시 "추상 메서드"
  - 인터페이스는 여러개 상속(?) 가능
  * class Bird implements Flyable, Walkable
